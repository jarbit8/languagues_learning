# languagues_learning — App web de inglés y francés en paralelo
Una única app web (PWA estática) para aprender inglés y francés a la vez, nivel por nivel (empezando por A1), a RITMO LIBRE basado en dominio, sin calendario. Se publica en GitHub Pages, uso desde el celular en https://jarbit8.github.io/languagues_learning/ (instalable). Meta: aprobar los exámenes de idiomas para la residencia canadiense (IELTS/CELPIP inglés, TEF/TCF Canada francés) — todos los exámenes internos imitan ese formato desde A1.

STACK (no negociable): Vite + React 18 + TypeScript + Tailwind CSS, SPA 100% estática — GitHub Pages no ejecuta servidor, PROHIBIDO backend/API routes/SSR. vite.config.ts con base: '/languagues_learning/' exacto al nombre del repo. Routing con HashRouter o tabs por estado, PROHIBIDO BrowserRouter (rompe al refrescar en Pages). PWA con vite-plugin-pwa, registerType autoUpdate. Todo el progreso en IndexedDB vía Dexie, sin backend ni auth. Audio con Web Speech API (speechSynthesis): inglés en-US (fallback en-GB), francés fr-FR (fallback fr-CA), rate 0.9. IA (speaking y corrección de writing): llamadas DIRECTAS del navegador a api.anthropic.com con header anthropic-dangerous-direct-browser-access: true; la API key la pega el usuario en Ajustes y vive SOLO en localStorage de su dispositivo, nunca en el código. Deploy automático: push a main → GitHub Actions → Pages.

SEGURIDAD (repo público): ningún secreto en código, commits ni historial. .gitignore desde el primer commit: node_modules, dist, .env*.

REGLAS DE PRODUCTO (no cambiar): 1) RITMO LIBRE — no hay días, fechas ni calendario; se avanza por dominio: estudiar tema → aprobar examen de tema → se desbloquea el siguiente. 2) El usuario marca cada palabra "aprendida" cuando decide; el examen diario evalúa EXACTAMENTE las que marcó ese día (sean 5 o 50) más los repasos SRS vencidos, sin mínimos ni máximos. 3) Cada concepto de vocabulario es UNA tarjeta dual con inglés + francés + audio de ambos, nunca cursos separados. 4) UI 100% en español, tono directo e informal. 5) Exámenes espejo de IELTS/TEF. 6) Arquitectura por data packs en /data, nada de contenido hardcodeado en componentes (así A2/B1/B2 entran como packs nuevos). 7) Ejercicios y diálogos solo usan vocabulario de temas ya desbloqueados. 8) La app funciona offline salvo speaking y corrección de writing.

ORDEN DE CONSTRUCCIÓN: Fase 0 scaffold+PWA+deploy. Fase 1 vocabulario+examen diario. Fase 2 gramática+examen de tema. Fase 3 contenido completo+examen de bloque+final. Fase 4 speaking+listening. Fase 5 progreso+pulido. Build sin errores antes de cada push.

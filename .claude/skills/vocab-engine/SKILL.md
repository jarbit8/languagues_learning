---
name: vocab-engine
description: Usar al construir o modificar el módulo de vocabulario - tarjetas duales, marcado libre, SRS, audio, formato JSON de packs.
---
PRINCIPIO: el usuario decide cuántas palabras aprende y cuándo. Sin cuota diaria ni presión de calendario; el sistema solo registra qué marcó y cuándo, para armar examen diario y repasos.
TARJETA (concepto dual): español + inglés + francés + audio por idioma + ejemplo A1 por idioma + nota opcional + pronunciación figurada opcional por idioma (campo `pron` dentro de en/fr). Esquema JSON: { "tema": N, "titulo": "...", "conceptos": [ { "id": "a1-tNN-NNN", "es": "...", "en": {"texto":"...","ejemplo":"...","pron":"opcional"}, "fr": {"texto":"...","ejemplo":"...","pron":"opcional"}, "nota": "opcional" } ] }. Genera los packs de los 24 temas con 30+ conceptos cada uno según el temario — el tamaño NO es un tope fijo, cada tema lleva las palabras de alta frecuencia reales de esa función aunque sean más de 30 (código dinámico, ver [[project-languagues-learning]]). Ejemplos SIEMPRE con gramática ya vista en temas anteriores o el actual.
PRONUNCIACIÓN FIGURADA (campo `pron`, 2026-07-18): debe leerse sola, palabra por palabra, SIN leyenda/clave general en la UI (se probó una leyenda colapsable arriba de la lista y el usuario la rechazó explícitamente — no reintroducirla). Convención sin símbolos raros, solo letras que un hispanohablante ya lee bien: sílabas separadas por guion, la sílaba tónica en MAYÚSCULAS. EN: «j» para la h aspirada (hello → je-LOU, sonido de jota), «sh» tal cual, «z» para el sonido th sordo (think → zink), «d» para el th sonoro (this → dis, aprovecha que la d intervocálica del español ya suena así). FR: «y» para el sonido de la j francesa (bonjour → bon-YUR), «u» para la u francesa (tu → TU, se simplifica aunque no sea 100% exacta), «e» para eu/œu (professeur → pro-fe-SER), nasales escritas tal cual con la n (bien → BYEN, sans → SAN). Todo esto reemplazó una versión anterior con «th/dh/zh/ü/ö» que si requería leyenda — no volver a esos símbolos.
FLUJO: usuario abre tema en curso → ve tarjetas con estado → toca una → escucha audios → botón "Aprendida ✓" (puede desmarcar el mismo día) → contador "hoy marcaste N palabras" alimenta el examen diario.
SRS 3 CAJAS: nueva→aprendida(timestamp)→en_repaso→dominada. Repaso a +1, +3, +7 días del marcado. Acierta las 3 → dominada. Falla en cualquier examen → vuelve a caja 1 (+1 día). Repasos vencidos se acumulan sin castigo ni mensajes de culpa.
AUDIO: SpeechSynthesisUtterance lang en-US (fallback en-GB) y fr-FR (fallback fr-CA), rate 0.9. Verificar voces con speechSynthesis.getVoices() (carga asíncrona, escuchar voiceschanged); si falta, aviso único con instrucciones de instalación. Reproducir también el ejemplo, no solo la palabra.
PERSISTENCIA (Dexie): palabras { id, estado, fechaAprendida, cajaSRS, proximoRepaso, aciertosSeguidos, fallosTotales }. El contenido se lee de los JSON; Dexie solo guarda estado.

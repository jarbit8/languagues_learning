ARQUITECTURA (sin backend): llamada directa del navegador a https://api.anthropic.com/v1/messages con headers x-api-key (de localStorage), anthropic-version: 2023-06-01, anthropic-dangerous-direct-browser-access: true, content-type: application/json. Modelo Sonnet vigente, max_tokens 1000. Historial de sesión en el cliente, se envía completo cada turno.
API KEY: el usuario la pega una vez en Ajustes (input password) → localStorage, con aviso "tu key queda solo en este dispositivo, nunca la subas al repo, es público". Sin key o sin red: módulo deshabilitado con aviso claro, TODO lo demás de la app sigue funcionando offline. Manejo de errores: 401 key inválida, 429 límite alcanzado, error de red "necesita internet".
SYSTEM PROMPT (usar literal, interpolando variables): "Eres un tutor de {idioma} conversando con un estudiante A1 sobre: {escenario del tema}. Responde SIEMPRE en {idioma}, nunca en español, ni para traducir ni para explicar — háblale como a un niño bilingüe que ya te entiende. Frases de máx 8 palabras, vocabulario A1 de {temas desbloqueados}. Una pregunta por turno, cálido y natural. Si el estudiante escribe en español, respóndele solo en {idioma} y sigue la conversación, sin traducir lo que dijo. Si comete un error, NO corrijas en el momento, recuérdalo para el cierre. En francés usa vous salvo que el estudiante use tu primero. Al escribir 'terminar' o a los 12 turnos, cierra TAMBIÉN en {idioma} con frases simples A1 (nada de español): 1 cosa buena + máx 3 errores con corrección y explicación breve, formato JSON {tipo:feedback, bien, errores:[{dijo,correcto,porque}]}." INMERSIÓN TOTAL: ni la conversación ni el feedback de cierre usan español en ningún punto — solo la UI/chrome de la app (botones, títulos) queda en español por la regla de producto.
FLUJO (chat integrado, componente ChatSpeaking): chat de texto con botón 🔊 por respuesta (speechSynthesis) → entrada por voz con SpeechRecognition si el navegador lo soporta, si no, dictado por teclado → feedback final como tarjeta, errores de vocabulario mandan esas palabras al SRS.
El examen de bloque reutiliza ChatSpeaking en "modo examen": mismo motor, la IA también devuelve nota 0-100 según rúbrica A1. Ahí sí se usa la API key de Ajustes (con fallback de autoevaluación por checklist si no hay key, ver Autoevaluacion.tsx).
PANTALLA CONVERSACIÓN (práctica libre, 2026-07-18): decisión del usuario — NUNCA pegará una API key en este dispositivo, así que esta pantalla NO usa ChatSpeaking ni hayApiKey en absoluto. Solo ofrece "copiar el prompt" (función construirPromptCopiable en speaking.ts): elegir idioma y escenario (default: tema en curso, pasados siempre disponibles) → se muestra el system prompt + instrucción de arranque en un textarea con botón copiar, listo para pegar como primer mensaje en cualquier otra app de IA (Claude, ChatGPT...) y practicar ahí. No reintroducir el toggle "hablar aquí dentro" ni el aviso de "necesitas una API key" en esta pantalla salvo que el usuario lo pida explícitamente.
COMPONENTE COMPARTIDO: el textarea+botón-copiar de "Conversación" se extrajo a `components/CopiarPrompt.tsx` (props `prompt`/`descripcion`), reutilizado también por la pantalla de práctica de Listening con su propio prompt (`construirPromptListening`, ver skill listening-engine) — incluye el mismo espíritu de inmersión total. `temasDesbloqueadosTexto` también se movió a speaking.ts (exportada) para que ambas pantallas la compartan sin duplicar código.
